## What You See in the ISO360 Tab (Customer Workspace)

When you open a customer's workspace and click the **ISO360** tab, here is what each section means:

---

### Hero Card (top banner)
The dark gradient card at the top shows the ISO standard name (e.g., "ISO 27001:2022").
- **Stats chips** — Total activities / Due Soon / Overdue / Completed This Year
- These are read-only status numbers computed from the database right now.

---

### Attention Strip (horizontal scroll row)
Orange/red cards that scroll horizontally — one per activity that is **due soon** or **overdue**.
- These are the things that need action first.
- Clicking a card opens the activity detail modal.
- **Right now:** These are informational only. No task exists in the system for them yet (Phase 4a not done).

---

### Activity Table (the main list)
All 33+ ISO360 activities for this customer, with:

| Column | What it shows |
|---|---|
| Type | Review / Operations / Record — what kind of compliance work it is |
| Activity | The personalized title of the activity |
| Frequency | Monthly / Quarterly / Yearly / Event-based |
| Next Due | The next date this activity must be completed by |
| Action | **Trigger button** (only on event-based rows) |

**Color coding on the left border of each row:**
- Red = Overdue
- Amber = Due soon
- Green = Completed
- Transparent = Upcoming / Event-based

**Filters:** You can filter by Type, Frequency, or search by name.

---

### Activity Detail Modal (click any row)
Opens a full detail panel showing:
- ISO clause reference (e.g., "5.23")
- Responsible role (e.g., "IT Security Manager")
- Step-by-step instructions — personalized for this customer by AI
- Evidence fields — what the customer needs to upload/fill (e.g., "Vendor contract signed", "Risk assessment result")

This is the personalized content generated by Phase 3 (adjustment agent). It reflects the customer's industry, answers, and context.

---

### Trigger Button (event-based rows only)
Event-based activities have no fixed schedule — they fire when something happens in the business (a new hire, a security incident, a new vendor, etc.).

The **Trigger** button (indigo, Zap icon) in the Action column:
- Clicking it **immediately creates a customer_task** in the database
- The task will appear in the customer's **Tasks tab** as a new open task
- The task has the ISO360 steps and evidence fields attached
- **If you click it twice**, you get a "Task already open" warning — it's idempotent

**Right now:** The task is created, but the customer does NOT yet receive a notification email (Phase 4a/scheduler handles that — not done yet).

## The Gap Between Now and "Live Service"

The single biggest missing piece is **Phase 4a — the scheduler**.

Without it:
- Monthly/quarterly/yearly activities never generate tasks automatically
- The customer never gets notified that something is due
- The ISO360 tab is a **view-only dashboard** — an admin can see what's due, but nothing reaches the customer

With Phase 4a:
- Every day at a configured time, a job scans for activities due within 7 days
- It creates a `customer_task` for each one found
- The customer sees the task in their portal with the steps and evidence fields
- The ISO360 tab becomes a **live compliance engine**

## Next Step: Phase 4a — The Scheduler

**What needs to be built:**

### 1. DB column
```sql
ALTER TABLE dna_app.customer_documents ADD COLUMN excluded BOOLEAN DEFAULT FALSE;
```
Activities with `excluded = TRUE` are skipped by the scheduler.

### 2. Scheduler job (automation-service/scheduler.py)
A daily job that:
```
Find all customer_documents WHERE:
  document_type = 'iso360_activity'
  AND update_frequency != 'event_based'
  AND next_due_date <= TODAY + 7
  AND last_completed_at IS NULL (or last_completed_at < next_due_date)
  AND excluded = FALSE
  AND no open customer_task already exists for this doc
→ Create customer_task (task_type='iso360_activity', source='iso360_scheduled')
→ Send notification email to customer
```

### 3. Task completion handler (backend)
When an ISO360 task is marked complete:
```
last_completed_at = NOW()
next_due_date = next_due_date + interval (1 month / 1 quarter / 1 year)
task status = 'completed'
```

### 4. Notification email
Reuse existing `send_campaign_email` with an ISO360-specific template:
"Your ISO 27001:2022 activity [title] is due in 7 days. Here are the steps..."
